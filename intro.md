# A/B Testing in Practice

Welcome to **A/B Testing in Practice** — a hands-on, project-based course where you learn to design, analyze, and communicate A/B test results like a product data scientist.

## The Story

You are a **product data scientist** at **FamilyNest** — think Airbnb, but for families with young kids. Your team is focused on growing the number of homes on the platform by improving the host onboarding flow.

Each week, you'll face realistic scenarios: designing experiments, catching validity issues, handling stakeholder pressure, and managing multiple concurrent tests.

## Course Structure

| Week | Topic | What You'll Learn |
|------|-------|-------------------|
| 1 | Designing A/B Tests | Metrics, hypotheses, power analysis, duration, launch decisions |
| 2 | Validity Threats | Assignment imbalance, targeting errors, segment analysis |
| 3 | Early Results | Peeking problem, novelty effects, stakeholder communication |
| 4 | Concurrent Tests | Multi-arm designs, interactions, factorial experiments |

## How to Use This Course

For each week:

1. **Read** the Study Notes — theory, formulas, and worked examples
2. **Attempt** the Assignment — scaffolded questions with hints (no answers!)
3. **Check** against the Solution — the original solved notebooks
4. **Self-assess** with the 30 Critical Questions

## Run the Notebooks

Click the **rocket icon** (🚀) at the top of any notebook page to open it in Google Colab, where you can run the code interactively.

Or clone the repo and run locally:
```bash
git clone https://github.com/vkoul/ab-test-practice.git
cd ab-test-practice
pip install -r requirements.txt
jupyter lab
```

## Prerequisites

- Python basics (pandas, numpy)
- Basic statistics (mean, variance, distributions)
- No prior A/B testing experience required — the study notes cover everything
